Cara Kerja HTTP dan HTTPS

Apa itu HTTPS?

Protokol Transfer Hiperteks Aman (HTTPS) adalah versi aman dari HTTP , protokol utama yang digunakan untuk mengirim data antara peramban web dan situs web. HTTPS dienkripsi untuk meningkatkan keamanan transfer data. Hal ini khususnya penting ketika pengguna mengirimkan data sensitif, seperti saat masuk ke rekening bank, layanan email, atau penyedia asuransi kesehatan.

Situs web apa pun, terutama yang memerlukan kredensial masuk, harus menggunakan HTTPS. Di peramban web modern seperti Chrome, situs web yang tidak menggunakan HTTPS ditandai secara berbeda dengan situs web yang menggunakan HTTPS. Cari ikon gembok di bilah URL untuk menandakan halaman web tersebut aman.

Ketika Anda mengunjungi situs web yang menggunakan HTTPS, koneksi antara browser Anda dan server situs web tersebut dienkripsi (artinya diacak) Ini melindungi data Anda dari mata-mata oleh penyerang.

Data tersebut mencakup semua jenis informasi rahasia—kredensial login, informasi pembayaran, dan aktivitas penelusuran secara umum. Jadi, dengan kata lain: Protokol HTTPS sangat penting untuk menjaga privasi dan keamanan data Anda saat menjelajah web.

Apa yang membedakan HTTPS dengan HTTP?

Secara teknis, HTTPS bukanlah protokol yang terpisah dari HTTP. HTTPS hanya menggunakan enkripsi TLS/SSL melalui protokol HTTP. HTTPS beroperasi berdasarkan transmisi sertifikat TLS/SSL , yang memverifikasi bahwa penyedia tertentu memang benar-benar penyedia yang mereka klaim.

Ketika pengguna terhubung ke halaman web, halaman web tersebut akan mengirimkan sertifikat SSL-nya yang berisi kunci publik yang diperlukan untuk memulai sesi aman. Kedua komputer, klien dan server, kemudian menjalani proses yang disebut jabat tangan SSL/TLS, yaitu serangkaian komunikasi bolak-balik yang digunakan untuk membangun koneksi aman.

Bagaimana cara kerja protokol HTTPS?

HTTP mengirimkan data yang tidak terenkripsi, yang berarti informasi yang dikirim dari peramban dapat dicegat dan dibaca oleh pihak ketiga. Proses ini kurang ideal, sehingga diperluas ke HTTPS untuk menambahkan lapisan keamanan tambahan pada komunikasi. HTTPS menggabungkan permintaan dan respons HTTP dengan teknologi SSL dan TLS.

Situs web HTTPS harus mendapatkan sertifikat SSL/TLS dari otoritas sertifikat (CA) independen. Situs web ini membagikan sertifikat tersebut dengan peramban sebelum bertukar data untuk membangun kepercayaan. Sertifikat SSL juga berisi informasi kriptografi, sehingga server dan peramban web dapat bertukar data terenkripsi atau acak. Prosesnya seperti ini:

  1. Anda mengunjungi situs web HTTPS dengan mengetikkan format URL https:// di bilah alamat peramban Anda.
  2. Peramban mencoba memverifikasi keaslian situs dengan meminta sertifikat SSL server.
  3. Server merespons dengan sertifikat SSL-nya, yang berisi kunci publik.
  4. Sertifikat SSL situs web membuktikan identitas server. Setelah peramban yakin, peramban menggunakan kunci publik untuk mengenkripsi dan mengirimkan pesan berisi kunci sesi rahasia.
  5. Server mendekripsi kunci sesi (private) untuk mendekripsi pesan dan mengambil kunci sesi. Kemudian, server mengenkripsi kunci sesi dan mengirimkan pesan konfirmasi ke peramban.
  6. Sekarang, browser dan server web berhasil menyalurkan kunci sesi yang sama untuk bertukar pesan yang aman.
